The Lists page (Admin > Advanced Settings > Lists) displays the that enables Administrators to maintain lists of information that can be identified by a key value and a data value. A related copy of the Lists module is also available on the Host > Advanced Settings > Lists page.

The Admin List module can access entries from Host List module, but Host List cannot access Admin List. The Host List stores site-wide Lists, such as Country, State, Region. Administrators can add new entries to lists maintained on the Host List menu as well as create new lists. =======

Lists, located on the Admin > Advanced Settings > Lists page, displays the that enables Administrators to maintain lists of information that can be identified by a key value and a data value. A related copy of the Lists module is also available on the Host > Advanced Settings > Lists page.

Admin List can access entries from Host List page, but Host List cannot access Admin List. The Host List stores site-wide Lists, such as Country, State, Region. Administrators can add new entries to lists maintained on the Host List menu as well as create new lists. >>>>>>> 72_release

The List module is based around Lists and Entries, which are in effect the same, but their usage defines their differences:

  • A List is a collection of Entries that has a Key Identifier and a Data Portion.
  • An Entry is an individual item of data that relates to a List and is made up of a Key Identifier and a Data Portion. An example of this is the countries list used in DNN. The list is 'Countries' and the entries are the actual countries. E.g. Australia, England, Holland

Note: Each Entry can also be a list. An example of this is the regions list used in DNN. In this case the Country, which is an entry under the Countries List, can be its own list and have region entries associated with it.
