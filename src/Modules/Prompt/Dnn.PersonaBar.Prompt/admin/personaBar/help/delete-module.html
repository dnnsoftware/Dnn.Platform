delete-module

Permanently deletes a module instance and all referenced instances or soft-deletes a module on a specific page. This will not uninstall modules or affect module definitions.

Options

Flag Value Required Default Value Description
--id integer Yes Explicitly specifies the Module ID of the module to retrieve. Use of the flag is not required. You can simply provide the ID value as the first argument.
--pageid integer No Specifies the Page ID of the page on which the module to delete resides. When you specify the pageid in addition to the required id, the module will be sent to the DNN Recycle Bin as opposed to being permanently deleted.

Permanently Delete a Module and All Its References

This will permanently delete the module with a Module ID of 345. If you have referenced this module from any other pages, they will also be deleted. NOTE: If you used the DNN user interface to "Add existing module" and checked the "Make a Copy", those module instances will not be deleted.
This is a permanent action and cannot be reversed.

delete-module 345

The following version is a more explicit version of the code above, but does the same thing.

delete-module --id 345

Delete and Send Module Instance to Recycle Bin

This will delete a module instance on a specific page and send it to the DNN Recycle Bin

delete-module 345 --pageid 42
